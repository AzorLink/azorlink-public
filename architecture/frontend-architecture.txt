DATA FLOW:
═════════
1. User interacts with a Component
2. Component uses a Hook for state/communication
3. Hook calls the API Client
4. API Client sends an HTTP request to the Backend
5. Response returns and updates state
6. Component re-renders with the new data

PRINCÍPIOS:
═══════════
✓ Component-based architecture
✓ Server-side rendering (SSR)
✓ Static site generation (SSG)
✓ Client-side interactivity
✓ Type safety with TypeScript
✓ Responsive design
✓ Accessibility (a11y)
