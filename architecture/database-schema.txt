RELATIONSHIPS:
═════════════
- Users 1:N Resources (one user can have many resources)
- Resources 1:N ResourceMetadata (one resource can have many metadata entries)
- Users 1:N Sessions (one user can have many sessions)

INDEXES:
═══════
- users.email (UNIQUE)
- resources.user_id
- resources.status
- sessions.token
- sessions.user_id
